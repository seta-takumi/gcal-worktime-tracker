# gcal-worktime-tracker 仕様書

## 1. 概要

Google カレンダーの予定をもとに、会議や休憩などを除いた 1 日の「作業可能時間」を自動計算し、次の 2 つの方法で可視化する Chrome 拡張機能。

1. **カレンダー上の終日予定**: 対象期間の各作業日に、その日の作業可能時間を示す終日イベントを自動作成（週単位で上書き更新）。
2. **ポップアップ**: 拡張アイコンをクリックすると、当日の「残り」作業可能時間を表示。

利用者は自分ひとり（本人のプライマリカレンダーのみを対象）。

---

## 2. 計算ロジック

### 2.1 就業時間帯

- **設定可能** な開始時刻・終了時刻（全日共通）。
- **デフォルト**: 10:00 – 19:00（9 時間）
- 設定値は options ページから変更し、`chrome.storage.sync` に保存する（端末間で同期）。
- 保存キー: `workingHours = { start: "HH:mm", end: "HH:mm" }`
- **バリデーション**: `start < end` を必須とする。範囲が不正な場合はデフォルト値で計算する。
- タイムゾーンはブラウザの現在タイムゾーンに従う。

### 2.2 集計単位

- **1 日単位**で作業可能時間を算出する。
- 週や月単位の合算表示はしない。

### 2.3 算出式

```
作業可能時間(d) = (就業時間帯の長さ) − Σ(就業時間帯と重なる予定の所要時間)
```

- **就業時間帯の長さ**: §2.1 の `end - start`（デフォルト設定なら 9h）。
- **対象予定**: プライマリカレンダーの、その日の `transparency != "transparent"`（= busy）かつ自分が **`responseStatus == "accepted"`（参加）** している予定。参加者なし（自分だけの予定）は常に対象。
- **就業時間帯との重なりのみカウント**: `start` より前に始まる会議なら `start` 以降の部分だけを差し引く。
- **予定の重複（二重計上防止）**: 同じ時間帯に複数の予定がある場合は時間帯をマージしてから合計する。
- **丸め**: 最終的な作業可能時間は **5 分単位**で切り下げ（表示用）。内部計算は分単位で保持する。
- **下限クランプ**: 結果が負になる場合は **0h** にクランプする。

### 2.4 除外・特殊ケース

| 条件 | 挙動 |
| --- | --- |
| タイトル小文字化して `lunch` を含む | **会議と同様に作業時間から差し引く（休憩扱い）**。参加判定はスキップし常に差し引く |
| タイトルに除外ワード（ユーザー設定）を含む | **計算から除外する（差し引かない）**。`lunch` と異なり休憩扱いではなく単に無視する |
| 同日に終日イベントがある（祝日・有給・OOO 等） | その日は**計算も書き込みもスキップ**。ただし `transparency == "transparent"` な終日イベントおよび `eventType == "workingLocation"`（勤務場所設定）はスキップ対象から除外する |
| 会議の合計が就業時間帯の長さを超過 | **0h にクランプ**して表示 |
| `transparency == "transparent"`（予定あり扱いしない設定の予定） | **差し引かない**（`lunch` キーワードを除く） |
| `responseStatus != "accepted"`（未承認・辞退の会議） | **差し引かない**（`lunch` キーワードを除く） |
| 終了済み（過去）の予定 | 計算対象（当日の残り時間計算では別途後述） |

### 2.5 当日の「残り作業可能時間」

ポップアップ表示用に、現在時刻以降に限定した再計算を行う。

```
残り作業可能時間 = max(0, (現在時刻〜end の就業時間帯残尺) − Σ(現在以降の会議 ∩ 就業時間帯))
```

- 現在時刻が `end` を過ぎていれば **0h**。
- 現在時刻が `start` より前なら、その日の作業可能時間と同値。

### 2.6 集計開始曜日

- **設定可能** な集計開始曜日（月〜金から選択）。
- **デフォルト**: 月曜日
- 設定値は options ページから変更し、`chrome.storage.sync` に保存する。
- 保存キー: `weekStartDay`（0=日〜6=土の整数）
- 作業日は集計開始曜日から 7 日間のうち**土日を除く 5 日間**を自動決定する。
  - 例: 集計開始曜日=水曜（3）→ 作業日は水・木・金・月・火
- スプリント期間が月曜始まり以外（水〜火など）の場合に利用する。

### 2.7 集計週数

- **設定可能** な集計週数（1〜4 週間）。
- **デフォルト**: 1 週間
- 設定値は options ページから変更し、`chrome.storage.sync` に保存する。
- 保存キー: `weekCount`（1〜4 の整数）
- 今週から weekCount 週分の作業日を対象とする。

### 2.8 対象期間（target period）の定義

終日予定の書き込みや API 取得範囲で参照する対象期間を、以下のとおり一元定義する。

- **週の起点**: §2.6 で設定した集計開始曜日。
- **作業日**: 集計開始曜日から 7 日間のうち土日を除く 5 日間（§2.6 参照）。
- **対象期間**: 今週（ロード時の現地日付が属する週）から §2.7 で設定した weekCount 週分。
- **書き込み対象日**:
  - **今週**: today 以降の作業日のみ書き込み・上書きの対象とする（today より前は除外）
  - **2 週目以降**: その週の全作業日を対象とする
- **API 取得範囲**: 各週の**初日 00:00 〜 最終日 23:59**（ブラウザタイムゾーン）を `timeMin` / `timeMax` に指定して週ごとに取得する。
- **過去日の扱い**: 書き込み対象外だが、削除も行わない（履歴として残す）。
- **作業日以外の当日分**: 作業日でない曜日（デフォルト設定では土日）はポップアップで「本日休日」表示とする。

## 3. 表示

### 3.1 カレンダー上の終日予定

- **作成範囲**: §2.7 で定義した **対象期間の書き込み対象日**（= today 以降の作業日）。
- **更新タイミング**: `calendar.google.com` をロードするたびに、対象期間の書き込み対象日を再計算して上書き。
- **過去日**: today より前の日は書き込みも削除もしない（履歴として保持）。
- **スキップ**: §2.4 で定めた「終日イベントがある作業日」は作成しない（既にある拡張作成イベントは削除する）。
- **タイトル書式**: `🧑‍💻 作業可能 6h30m`
  - `h` と `m` の後ろにスペースなし。分が 0 の場合は `🧑‍💻 作業可能 7h`。
  - 0h の場合は `🧑‍💻 作業可能 0h`。
- **識別方法**: Google Calendar API の `extendedProperties.private` に以下を付与し、拡張作成物かを判定する。
  ```json
  {
    "extendedProperties": {
      "private": {
        "workHoursExtension": "v1"
      }
    }
  }
  ```
- **タイトルベースの判定はしない**: ユーザーが手動でタイトルを変更しても上書き動作を壊さないため。

### 3.2 ポップアップ

- 拡張アイコンをクリックして開く。
- **表示内容**: 当日の **残り作業可能時間のみ**（例: `残り 3h15m`）。
- **更新タイミング**: ポップアップを開いた時点で最新のカレンダー情報を取得して計算。
- 就業時間帯の `end` 以降は `本日終了` などの文言で非アクティブ表示（詳細はデザイン時に詰める）。
- オプション画面へのリンクをフッターに配置する。

### 3.3 カレンダーページ上のオーバーレイ

- Google カレンダーのヘッダー付近に、当日の残り作業可能時間を小さくピン留め表示。
- 表示内容はポップアップと同様（当日の残り時間）。
- 当日分のみ表示し、週全体は出さない（視認性重視）。
- 具体的な DOM マウント位置・スタイルは実装時に決定。

## 4. データ連携 / 認証

- **API**: Google Calendar API v3
- **認証**: `chrome.identity.getAuthToken({ interactive: true })` で OAuth2 アクセストークン取得
- **スコープ**: `https://www.googleapis.com/auth/calendar.events`（読み書き）
- **対象カレンダー**: プライマリカレンダー（`calendarId = "primary"`）のみ
- **取得範囲**: §2.8 の対象期間（各週の初日 00:00 〜 最終日 23:59、ブラウザタイムゾーン）を `timeMin` / `timeMax` に指定した `events.list`。`singleEvents=true`, `orderBy=startTime` で繰り返し予定を展開。週ごとに個別にリクエストを送る。

## 5. 動作トリガー

| きっかけ | 動作 |
| --- | --- |
| `calendar.google.com` をロード（作業日） | 今週から weekCount 週分の書き込み対象日を再計算 → 終日予定の上書き、オーバーレイ更新 |
| `calendar.google.com` をロード（作業日以外） | 今週から weekCount 週分の終日予定を上書き。オーバーレイ / ポップアップは「本日休日」表示 |
| SPA 内ナビゲーション（URL 変化） | MutationObserver または `popstate` で検知し再実行 |
| 拡張アイコンクリック（ポップアップ開く） | 当日分の残り時間を再計算して表示（作業日以外は「本日休日」） |
| バックグラウンドの定期ポーリング | **行わない**（ロード時計算のみ） |

---

## 6. エラーハンドリング / エッジケース

- **認証失敗**: ポップアップに再認証ボタン、オーバーレイは非表示。
- **API レート制限**: 指数バックオフで最大 3 回リトライ。失敗した場合は前回計算結果（ローカルキャッシュ）を表示し、エラー状態を UI に出す。
- **ネットワーク断**: 直近のキャッシュを表示し、ステータス表示で古いデータである旨を示す。
- **タイムゾーン不一致**: イベントのタイムゾーンはブラウザ側のタイムゾーンに正規化してから重なり計算。
- **終日予定の重複作成防止**: 同日に `workHoursExtension=v1` のイベントが複数見つかった場合、1 件だけ残して他は削除する。

## 7. 拡張構成（予定）

- **Manifest V3**
- **コンポーネント**:
  - `background (service worker)`: 認証・API 呼び出し・計算の中枢
  - `content script`: `calendar.google.com` に注入。オーバーレイ描画と SPA 遷移検知
  - `popup`: 当日残り時間の表示
  - `options`: 就業時間帯（`start` / `end`）、集計開始曜日（`weekStartDay`）、集計週数（`weekCount`）、除外ワード（`excludeKeywords`）の設定 UI
- **ストレージ**:
  - `chrome.storage.sync`: ユーザー設定（`workingHours`, `weekStartDay`, `weekCount`, `excludeKeywords`）
  - `chrome.storage.local`: 直近計算結果のキャッシュ
- **設定変更時の挙動**: `chrome.storage.onChanged` を購読し、設定変更を検知したら次回ロード時に再計算（即時再計算はしない）。
